# Meta Starter Is My Snowball

Before AI, documenting experience was painful. I had to organize it or it was useless: disorganized notes are just noise when I need to find something later. The overhead of organizing killed the habit.

## The pattern

Very small investment: put thoughts on how to do certain things, these become patterns.

The payoff: when I need to reuse a pattern in another project, point an AI agent to the meta-starter repo (wired in via `/meta-starter` skill) and ask it to implement the thing based on the pattern.

The result is a collection of patterns that are low-maintenance (update when I need to) but high-ROI because using them is as easy as talking to an agent.

## The snowball

```
more projects
  → more experience
    → more things to add to meta-starter
      → implement more projects faster
        → more projects
```

Each project feeds the next. The patterns compound.

## Why it works

- **No organization overhead**: AI navigates the patterns, not me. Rough notes are fine. It's all just markdown files!
- **Zero-friction reuse**: describe what I want, reference the pattern, agent does the rest.
- **Self-updating**: I add to it when I encounter something worth keeping, not on a schedule.
- **Cross-project**: one pattern repo serves every project I work on.

The meta-starter is not a template. It is a living record of how I work, queryable by an agent that knows my codebase.
